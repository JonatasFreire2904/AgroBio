Metodologia de Desenvolvimento

Abordagem Ágil e Desenvolvimento Iterativo

1. Metodologia Ágil

O desenvolvimento da plataforma AgroBio seguirá os princípios da metodologia ágil, priorizando a entrega incremental de funcionalidades, colaboração entre equipe e adaptação a mudanças durante o processo de desenvolvimento.

1.1 Princípios Aplicados

Indivíduos e Interações

Valorizamos a comunicação direta e colaboração entre os membros da equipe

Software Funcionando

Entregamos software funcional em intervalos regulares

Colaboração com Cliente

Envolvemos os usuários finais no processo de desenvolvimento

Responder a Mudanças

Adaptamos o projeto conforme feedback e necessidades emergentes

2. Ciclo de Desenvolvimento

2.1 Sprints de Desenvolvimento

O projeto será dividido em sprints de 2 semanas, cada um focado em entregar funcionalidades específicas e testáveis.

Sprint 1

Configuração e Autenticação

  • Configuração do ambiente de desenvolvimento
  • Implementação do sistema de autenticação
  • Criação do banco de dados
  • Interface de login e cadastro
Sprint 2

Gestão de Usuários

  • Sistema de perfis diferenciados
  • Atualização de dados do usuário
  • Dashboard personalizado
Sprint 3

Sistema de Ofertas

  • Publicação de ofertas de resíduos
  • Busca e filtros de ofertas
  • Visualização de detalhes
Sprint 4

Sistema de Negociações

  • Iniciação de negociações
  • Chat entre usuários
  • Status de negociações

4. Arquitetura do Sistema

4.1 Arquitetura em Camadas

Apresentação (Frontend)

Interface do usuário, formulários, dashboards

Aplicação (Backend)

Lógica de negócio, autenticação, validações

Dados (Database)

Armazenamento de usuários, ofertas, negociações

4.2 Padrões de Design

  • MVC (Model-View-Controller): Separação clara de responsabilidades
  • RESTful API: Endpoints padronizados e semânticos
  • Middleware Pattern: Processamento de requisições em camadas
  • Repository Pattern: Abstração do acesso a dados
